  整体区别

  - .editorconfig：跨编辑器的基础排版规范，管缩进、换行符、文件末尾换行、尾随空格等。

  - shfmt / .shfmt：Shell 脚本格式化工具；严格说 shfmt 官方主要用命令行参数和 .editorconfig
    配置，不是标准依赖 .shfmt。

  - .shellcheckrc：ShellCheck 的静态检查配置，管 Shell 脚本中的潜在 bug、可移植性、引用问题
    等。

  - markdownlint 配置：Markdown 文档规范检查，管标题、列表、空行、行长、代码块语言等。

  .editorconfig
  .editorconfig 用来让不同编辑器/IDE 在同一个项目里遵循一致的基础格式。EditorConfig
  官方说明它会从当前文件目录向上查找 .editorconfig，遇到 root = true
  停止；规则自上而下读取，越近的配置优先。常见属性包括
  indent_style、indent_size、end_of_line、charset、trim_trailing_whitespace、insert_final_ne
  wline。来源：EditorConfig 官方文档 (editorconfig.org (https://editorconfig.org/))

  常见示例：

  # .editorconfig
  root = true

  [*]
  charset = utf-8
  end_of_line = lf
  indent_style = space
  indent_size = 2
  trim_trailing_whitespace = true
  insert_final_newline = true

  [Makefile]
  indent_style = tab

  [*.md]
  trim_trailing_whitespace = false

  用法：

  - 放在项目根目录。
  - 编辑器打开文件时自动读取；VS Code、JetBrains、Vim/Neovim 等通常都有内置或插件支持。
  - 适合定义“全项目通用底线”，例如 LF 换行、UTF-8、末尾换行、2 空格缩进。
  - 对 Shell 来说，shfmt 也能读取 .editorconfig 中的 Shell 专用配置。

  shfmt / .shfmt
  shfmt 是 Shell 脚本格式化工具，用来格式化 sh、bash、mksh、bats 等脚本。它支持 -w 写回、-d
  输出 diff、-l 列出需要格式化的文件、-i 设置缩进、-ci 设置 case 缩进、-bn 允许 &&/|
  放到下一行开头等。官方手册也说明：如果发现 EditorConfig
  文件，会用其中选项；但如果命令行传了 parser/printer flags，则不会使用 EditorConfig
  格式选项。来源：shfmt 手册 (manpages.ubuntu.com
  (https://manpages.ubuntu.com/manpages/questing/man1/shfmt.1.html))

  常用命令：

  # 格式化单个文件，写回
  shfmt -w script.sh

  # 检查当前目录下哪些 Shell 文件格式不一致
  shfmt -l .

  # CI 中只输出 diff，不改文件
  shfmt -d .

  # 使用 Google Shell Style 类似风格
  shfmt -i 2 -ci -bn -w .

  更推荐写进 .editorconfig：

  # .editorconfig
  [*.sh]
  indent_style = space
  indent_size = 2
  shell_variant = bash
  binary_next_line = true
  switch_case_indent = true
  space_redirects = true

  关于 .shfmt：

  - shfmt 官方常规配置方式不是 .shfmt，而是命令行参数或 .editorconfig。
  - 有些项目会自定义 .shfmt，例如里面写 -i 2 -ci -bn，然后脚本这样调用：

  shfmt $(cat .shfmt) -w .

  - 这种 .shfmt 是团队约定，不一定被编辑器、CI、shfmt 本身自动识别。为了可移植，优先用
    .editorconfig 或在 package.json/Makefile/CI 脚本中固定参数。

  .shellcheckrc
  .shellcheckrc 是 ShellCheck 的配置文件。ShellCheck 是 Shell 脚本静态分析和 lint
  工具，不负责格式化，主要发现 bug、危险写法、不兼容写法、变量引用问题等。ShellCheck
  官方说明它会在脚本目录和父目录中查找 .shellcheckrc 或
  shellcheckrc，没找到再找用户级配置；文件内容是 key=value 形式，并会作为文件级 directive
  处理。来源：ShellCheck 手册 (raw.githubusercontent.com
  (https://raw.githubusercontent.com/koalaman/shellcheck/master/shellcheck.1.md))

  常见示例：

  # .shellcheckrc

  # 按 bash 语法检查；适合无 shebang 的脚本片段
  shell=bash

  # 允许 ShellCheck 跟踪 source 的外部文件
  external-sources=true

  # source 路径相对当前脚本目录解析
  source-path=SCRIPTDIR

  # 启用可选检查
  enable=check-unassigned-uppercase
  enable=quote-safe-variables

  # 禁用某些规则
  disable=SC1091
  disable=SC2034

  常用命令：

  # 检查单个脚本
  shellcheck script.sh

  # 检查多个脚本
  shellcheck scripts/*.sh

  # 跟踪 source 引入的文件
  shellcheck -x script.sh

  # 指定 shell 方言
  shellcheck --shell=bash script.sh

  # 生成可应用的修复 diff；不是所有问题都能自动修
  shellcheck --format=diff script.sh

  也可以在脚本里局部忽略规则：

  # shellcheck disable=SC2086
  echo $value

  建议：

  - 能修就修，不要轻易 disable。
  - 禁用规则时最好写注释说明原因。
  - .shellcheckrc 更适合放项目根目录，让 CI、本地编辑器、pre-commit 统一行为。

  markdownlint 配置
  markdownlint 用来检查 Markdown/CommonMark
  文档风格一致性。它会检查标题层级、列表缩进、空行、尾随空格、代码块语言、裸
  URL、重复标题等。官方 markdownlint 项目说明它是一个 Markdown/CommonMark
  静态分析工具，默认有一系列规则；VS Code 插件和 CLI 都能读取项目配置。来源：markdownlint
  官方仓库 (github.com (https://github.com/DavidAnson/markdownlint))

  常见配置文件名：

  - .markdownlint.json
  - .markdownlint.jsonc
  - .markdownlint.yaml
  - .markdownlint.yml
  - .markdownlintrc
  - .markdownlint-cli2.jsonc
  - .markdownlint-cli2.yaml
  - .markdownlint-cli2.cjs
  - .markdownlintignore：忽略某些 Markdown 文件

  VS Code markdownlint 文档说明，可用
  .markdownlint.jsonc/.json、.markdownlint.yaml/.yml、.markdownlint.cjs
  等配置；并且项目本地配置优先于 VS Code 用户/工作区设置。来源：vscode-markdownlint 文档
  (github.com (https://github.com/DavidAnson/vscode-markdownlint))

  .markdownlint.json 示例：

  {
    "default": true,
    "MD013": false,
    "MD033": false,
    "MD007": {
      "indent": 2
    },
    "MD003": {
      "style": "atx"
    }
  }

  .markdownlint.yaml 示例：

  default: true

  MD013: false

  MD007:
    indent: 2

  MD003:
    style: atx

  常用命令：

  # 检查所有 Markdown 文件
  markdownlint "**/*.md"

  # 指定配置文件
  markdownlint --config .markdownlint.json "**/*.md"

  # 自动修复可修复的问题
  markdownlint --fix "**/*.md"

  markdownlint-cli 文档说明，--fix 会尝试应用可修复规则，但不是所有问题都能自动修；未指定
  --config 时，会查找 .markdownlint.jsonc/.json/.yaml/.yml 或
  .markdownlintrc。来源：markdownlint-cli 文档 (github.com
  (https://github.com/igorshubovych/markdownlint-cli))

  文档内也可以临时关闭规则：

  <!-- markdownlint-disable MD013 -->

  这一段可以超过行长限制。

  <!-- markdownlint-enable MD013 -->

  推荐组合

  - .editorconfig：全项目基础格式，所有语言都受益。
  - .editorconfig + shfmt：统一 Shell 脚本排版。
  - .shellcheckrc + shellcheck：检查 Shell 脚本质量和 bug。
  - .markdownlint.json 或 .markdownlint.yaml：统一 Markdown 文档风格。
  - CI 可这样跑：

  shfmt -d .
  shellcheck scripts/*.sh
  markdownlint "**/*.md"

  一句话总结：.editorconfig 管“编辑器基础排版”，shfmt 管“Shell 自动格式化”，.shellcheckrc
  管“Shell 静态检查规则”，markdownlint 管“Markdown 文档规范”。
