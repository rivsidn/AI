
## 软件需求

该软件能在已固化好的几个拓扑图中，编排进自己的设备，从而形成一套完整的
组网环境。

主要是针对网络软件开发者使用，基于该软件可以快速搭建组网环境，用于:

- 开发过程中功能验证
- 集成测试等


### 用户接口

#### 拓扑配置

用户第一次使用的过程中必须能看到详细的组网拓扑图，需要明显标记出用户设备
在拓扑中所在的位置。

用户设备是需要用配置的，不能为空，不设置默认。
其他位置设备选配。

#### 配置文件

拓扑配置之后可以通过配置文件的方式保存当前配置，后续使用的时候可以直接通过
配置文件启动拓扑，不需要重复配置。

## 概要设计

软件基于'vrnetlab'、'containerlab' 实现，更底层技术用的是'docker'、'qemu'。

```
                        +--------+
       +--------------> |构建镜像| -------------+
       |                +--------+              v
       |                                   +----------+
       |                    +--------------|Docker镜像|
       |                    |              +----------+
    +------+            +---v----+                     
    |命令行|----------> |配置拓扑| -------------+
    +------+            +--------+              v
       |                                   +------------+
       |                    +--------------|拓扑配置文件|
       |                    |              +------------+
       |                +---v----+
       +--------------> |启动拓扑|
                        +--------+
```

如上图示.

命令行作为入口点，可以分别用于 构建镜像、配置拓扑、启动拓扑。

构建镜像会生成 Docker镜像。

配置拓扑依赖于上一步生成的 Docker镜像，配置拓扑会生成拓扑配置文件。

利用生成的拓扑配置文件启动拓扑。

### 命令行设计

#### 构建镜像

```bash
vrnet-topology --create-docker-image { switch|router|client|server }

vrnet-topology --create-docker-image local --image xxx.img
```

参数说明.

| 选项                  | 说明            | 参数             |
|-----------------------|-----------------|------------------|
| -c                    | 创建docker 镜像 | 指定镜像类型     |
| --create-docker-image | 创建docker 镜像 | 指定镜像类型     |
| -i                    | 本地镜像文件    | 指定本地镜像地址 |
| --image               | 本地镜像文件    | 指定本地镜像地址 |

- switch、router、client、server 为工具自带的默认镜像，不需要指定镜像文件
- local为本地镜像，需要指定镜像文件

#### 配置拓扑

```bash
vrnet-topology --setting-network-topoloty "name"
```

| 选项                       | 说明         | 参数               |
|----------------------------|--------------|--------------------|
| -s                         | 配置网络拓扑 | 指定创建的拓扑名称 |
| --setting-network-topoloty | 配置网络拓扑 | 指定创建的拓扑名称 |

可以指定拓扑名称，拓扑名称会保存在配置文件中，显示时使用。

- 拓扑配置结束之后可以指定保存配置文件，将配置文件保存在特定位置
- 可以选择是否指定配置文件为默认

#### 启动拓扑

```bash
vrnet-topology --deploy-network-topoloty --topology-config-file xxx.yml
```

| 选项                      | 说明         | 参数         |
|---------------------------|--------------|--------------|
| -d                        | 部署网络拓扑 | 空           |
| --deploy-network-topoloty | 部署网络拓扑 | 空           |
| -f                        | 指定拓扑文件 | 拓扑配置文件 |
| --topology-config-file    | 指定拓扑文件 | 拓扑配置文件 |

- 可以指定配置文件，启动对应的拓扑
- 不指定时启动默认配置，如果默认配置不存在则提示错误

#### 显示拓扑

```bash
vrnet-topology --list-network-topoloty
```

| 选项                    | 说明             |
|-------------------------|------------------|
| -l                      | 显示所有网络拓扑 |
| --list-network-topoloty | 显示所有网络拓扑 |

由于可以同时启动多个拓扑，需要配置name 显示。

### 构建镜像

构建镜像可以参考`vrnetlab` 实现，

主要包含这几部分:

- Dockerfile

  用于构建Docker 镜像.

  由于有多种镜像，可能包含多个不同用途的 Dockerfile.

- 启动脚本

  用作Docker 的`ENTRYPOINT`.


### 配置拓扑

配置拓扑会有一个模板文件，仅仅支持内置几种拓扑的配置。

配置拓扑最终会形成`containerlab` 中的配置文件。

#### 拓扑模板

```yml
name: client-server
topology:
  nodes:
    client:
      kind: generic_vm
      image: ${NEED-CONFIG}
    server:
      kind: generic_vm
      image: ${NEED-CONFIG}
  links:
    - endpoints: ["node1:eth2", "node2:eth2"]
```
